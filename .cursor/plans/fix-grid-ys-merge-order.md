# План: порядок GridYs при merge слоёв сетки

**Статус:** реализовано (2026-06-08)

## Проблема (лог 35NK)

```
[POSC] Сетка таблицы: линии на разных слоях — оси дополнены (0_ГТ-20_Изм.1 + ГС_ВК).
[POSC] Марок в данных (как BindKeys) по столбцам: col0=0, col1=0, … col5=0 (ColMark=0)
```

Подсчёт совпал с BindKeys (везде 0), но марки в таблице есть. Причина: после `MergeAxisClusters` ось Y шла **снизу вверх** (возрастание), а `CellIndex.TryGetCellIndex` ожидает **сверху вниз** (убывание). Все тексты получали `Row=-1`.

## Решение

`MergeAxisClusters(primary, extra, sortAsc)` — для X `sortAsc: true`, для Y `sortAsc: false`; кластеризация с тем же направлением.

## Проверка

1. `NETLOAD` свежей DLL (в логе часто `LOAD failed: pos_counter_2016_2026` — нужен ручной NETLOAD).
2. Чертёж 35NK, 2 таблицы.
3. Ожидаемо: counts > 0 в столбце с позициями; нет «Заполните колонку «Марка»»; запись в «Кол.» работает.

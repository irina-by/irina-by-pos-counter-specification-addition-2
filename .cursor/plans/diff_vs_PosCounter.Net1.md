# Отличия текущей версии от эталона PosCounter.Net1

**Эталон (старая рабочая сборка):** `C:\Users\user\Desktop\1\PosCounter.Net1`  
**Текущий код:** `PosCounter.Net` в репозитории  
**Дата сравнения:** 2026-06-15  
**Версия:** 4.2.0-table-grid-lines

---

## Краткий итог

| Область | PosCounter.Net1 (старая) | Сейчас |
|---------|--------------------------|--------|
| Подсчёт выносок (`PosCounterEngine`) | эталон | **тот же файл** (идентичен) |
| Спецификация (`TableGrid`, `SpecGridService`) | эталон | **те же файлы** (идентичны) |
| Палитра UI (`PosCounterControl`) | эталон | **тот же файл** (идентичен) |
| Запись «Кол.» | сумма по **всем** строкам `_lastCountRows` (в т.ч. скрытым фильтром) | сумма только по **видимым** строкам палитры |
| `CalloutMarkGate` (фильтры выносок C1–C4) | нет | **нет** (было в промежуточной версии — убрано) |
| `ProxyEntityHelper` (разбор СПДС proxy) | нет | **нет** (было — убрано) |
| Сборка | скрипт `build-ac2016.cmd` | **только Visual Studio** (скрипты `.cmd` удалены) |
| AutoCAD 2026 | отдельная сборка net8 | `dll 2026\` в корне репо |

**Главное отличие для инженера:** в «Кол.» на чертеже пишется количество **как в видимой таблице палитры**, а не «из памяти» по скрытым строкам.

## 2. Единственное изменение в коде распознавания/записи

### `PaletteHost.cs` — откуда берётся количество для «Кол.»

| | PosCounter.Net1 | Сейчас |
|---|-----------------|--------|
| Метод | `TryBuildQtyByKeyFromAllRowsSnapshot()` | `TryBuildQtyByKeyFromVisibleRows()` |
| Источник данных | `_lastCountRows` — все строки после ЗАПУСТИТЬ | `GetVisibleDataRows()` — только строки, **видимые в палитре** (с учётом фильтров Марка/Слой/Кол.) |
| Симптом старой версии | В палитре видно 2, в «Кол.» пишется 5 (сумма по скрытым слоям) | В «Кол.» пишется то же, что видно в колонке «Количество» |
| После **Сброс** | палитра пустая, но `_lastCountRows` могли остаться до сброса; при выборе спецификации без ЗАПУСТИТЬ — риск старых цифр | после Сброс `qtyByKey` пустой → **ничего не записывается** |

Место вставки на чертеже, стиль текста, столбец ColQty — **не менялись** (`SpecGridService.WriteQtyScope` без изменений).

## 3. Что было в промежуточных версиях и убрано (сейчас снова как Net1)

| Функция | Было добавлено | Статус сейчас |
|---------|----------------|---------------|
| `CalloutMarkGate.cs` | фильтр выносок (треугольник, круг, соседний текст) | **удалён** — подсчёт как Net1 |
| `ProxyEntityHelper.cs` | Explode proxy СПДС | **удалён** — proxy не читается |
| PickFirst через `ExecuteInApplicationContext` | выделение на листе | **убрано** — как Net1 |
| `StatusHint` для proxy | подсказка в палитре | **убрано** |

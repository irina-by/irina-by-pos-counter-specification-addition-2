# Сборка PosCounter.Net для AutoCAD 2016 SP1+ (портативный набор)

Эта папка — **автономный набор** для сборки на рабочем ПК. Скопируйте **всю** папку `PosCounter.Net` на рабочий стол (папки `bin\` и `obj\` можно не копировать — они создадутся при сборке).

## Что нужно на компьютере

- AutoCAD **2016 x64** (в т.ч. **SP1**, R20.1, .NET Framework **4.5.2** — сборка M.49.0)
- Visual Studio 2019/2022 **или** [.NET SDK](https://dotnet.microsoft.com/download)
- .NET Framework **4.5.2** Targeting Pack (обычно ставится с VS; на ПК с AC 2016 SP1 уже есть)

## Шаг 1. Путь к AutoCAD

Файл `build\AutoCAD.props` уже настроен:

```xml
<AutoCADSdkDirNet46>C:\Program Files\Autodesk\AutoCAD 2016\</AutoCADSdkDirNet46>
```

Если папка AutoCAD называется иначе (например «AutoCAD 2016 - Русский»), откройте `build\AutoCAD.props` в блокноте и укажите **папку**, где лежит `AcMgd.dll` (со слэшем в конце).

Найти папку в PowerShell:

```powershell
Get-ChildItem 'C:\Program Files\Autodesk' -Recurse -Filter AcMgd.dll -ErrorAction SilentlyContinue | Select-Object -First 3 FullName
```

## Шаг 2. Сборка в Visual Studio

1. Откройте `PosCounter.Net.csproj` в Visual Studio.
2. Меню **Проект** → **Восстановить пакеты NuGet** (или правый клик по решению → Restore).
3. Вверху выберите:
   - конфигурация: **Release**
   - платформа: **x64**
   - целевой framework: **net452**
4. **Сборка** → **Собрать проект**.

Готовые файлы:

```
PosCounter.Net\bin\x64\Release\net452\PosCounter.Net.dll
PosCounter.Net\bin\x64\Release\net452\System.ValueTuple.dll
```

## Шаг 3. Сборка из командной строки (проще)

Двойной клик по файлу:

```
build\build-ac2016.cmd
```

Скрипт соберёт проект и скопирует оба DLL в папку `dll 2016\`.

Окно командной строки закроется само, если сборка прошла без ошибок. Если что-то пошло не так — прочитайте текст в окне до закрытия или запустите скрипт из уже открытой cmd (перетащите файл в окно cmd).

### Сообщение про «телеметрию» (Telemetry)

При первом запуске `dotnet` Microsoft может показать текст вроде:

> *Инструменты .NET собирают данные об использовании… Вы можете отказаться, установив переменную DOTNET_CLI_TELEMETRY_OPTOUT…*

**Это не наша программа** — это стандартное уведомление от **.NET SDK** (установленного на компьютере). На сборку плагина и AutoCAD **не влияет**. Можно просто прочитать и продолжить.

В скрипте `build-ac2016.cmd` телеметрия уже отключена (`DOTNET_CLI_TELEMETRY_OPTOUT=1`), чтобы это сообщение не мешало.

Если хотите отключить телеметрию для всех команд `dotnet` на этом ПК навсегда — в PowerShell (от администратора не нужно):

```powershell
[System.Environment]::SetEnvironmentVariable('DOTNET_CLI_TELEMETRY_OPTOUT', '1', 'User')
```

## Шаг 4. Загрузка в AutoCAD 2016

В папке сборки или `dll 2016\` должны лежать **два** файла:

- `PosCounter.Net.dll`
- `System.ValueTuple.dll`

В AutoCAD:

1. Команда **NETLOAD**
2. Выберите свежую `PosCounter.Net.dll` из `bin\x64\Release\net452\` (рекомендуется) или из `dll 2016\` после `build-ac2016.cmd`
3. В командной строке: `[POSC] PosCounter.Net … (net452) build=ГГГГ-ММ-ДД ЧЧ:ММ загружен`
4. При «Выбрать спецификацию» первая строка `[POSC-DIAG]` тоже содержит `build=` — даты должны совпадать

## Частые ошибки

| Ошибка | Что сделать |
|--------|-------------|
| `AcMgd.dll not found` | Проверьте путь в `build\AutoCAD.props` |
| `System.ValueTuple 4.5.0 не найден` / NuGet restore | В проекте есть запасная DLL: `lib\netstandard1.0\System.ValueTuple.dll` — **перекопируйте** папку `PosCounter.Net` с Yandex целиком. В VS: **Сборка → Очистить** → **Собрать**. Или: `dotnet restore` затем `build-ac2016.cmd`. Короткий путь (не глубокий OneDrive) помогает restore. |
| NETLOAD в AC 2016 падает на ValueTuple | Положите **оба** DLL в одну папку (`dll 2016\`) |
| Старый LISP `pos_counter_…` | Не используйте — только **NETLOAD** .NET DLL |
| **`PaletteHost` уже содержит определение** / дубли методов | На рабочем ПК **два** файла `PaletteHost` или файл склеен дважды — см. ниже |
| **`SpecGridService` / `SpecPickResult` уже содержит определение** | Два файла `SpecGridService*.cs` или `SpecGridService.cs` вставлен дважды / в конец `TableGrid.cs` — см. ниже |

### Ошибка «PaletteHost уже содержит определение»

На Yandex в проекте **один** файл `PosCounter.Net\PaletteHost.cs`. Ошибка появляется, если при копировании на рабочий ПК папки **наложились** друг на друга.

**Быстрая проверка (PowerShell на рабочем ПК):**

```powershell
cd "C:\Users\Salnikava.I\OneDrive\Документы\программ раб комп\Pos_counter addition — 2\PosCounter.Net"
powershell -ExecutionPolicy Bypass -File .\build\verify-no-duplicate-sources.ps1
```

Должно быть: `[OK] PaletteHost + SpecGridService + TableGrid checks passed`.

**Вручную:**

```powershell
Get-ChildItem -Recurse -Filter "PaletteHost*.cs" | Where-Object { $_.FullName -notmatch '\\obj\\|\\bin\\' } | Select-Object FullName
```

Должен быть **ровно один** путь, например:
`...\PosCounter.Net\PaletteHost.cs`

В файле — комментарий `POSC-SINGLE-FILE: PaletteHost` и **~657 строк**.  
Если видите два файла (`PaletteHost.cs` и `PaletteHost (1).cs`) — **удалите лишний**.

**В Visual Studio:** Обозреватель решений → проект `PosCounter.Net` → файл `PaletteHost.cs` должен быть **один**. Если два — удалите лишний из проекта (ПКМ → исключить).

**Проверка внутри файла:** откройте `PaletteHost.cs` → поиск `public static class PaletteHost` — должно быть **1** вхождение. Если **2** — файл случайно вставили дважды; оставьте одну копию класса (до строки `}` закрывающей namespace).

**Чистая перекопировка (рекомендуется):**

1. На рабочем ПК **переименуйте** старую папку `PosCounter.Net` в `PosCounter.Net_old`.
2. Скопируйте **целиком** свежую `PosCounter.Net` с Yandex (без `bin\` и `obj\`).
3. Скопируйте из `_old` только `build\AutoCAD.props` (путь к AutoCAD 2016), если там уже был правильный путь.
4. Удалите папки `bin` и `obj`, если появились.
5. VS: **Сборка** → **Очистить решение** → **Собрать** (Release, x64, net452).

### Ошибка «SpecGridService / SpecPickResult уже содержит определение»

Та же причина, что у `PaletteHost`: при копировании папки на рабочий ПК файл **склеился** или появился **второй** `SpecGridService*.cs`.

**Проверка:** `build\verify-no-duplicate-sources.ps1` (запускается автоматически из `build-ac2016.cmd`).

**Вручную:**

```powershell
Get-ChildItem -Recurse -Filter "SpecGridService*.cs" | Where-Object { $_.FullName -notmatch '\\obj\\|\\bin\\' } | Select-Object FullName
```

Должен быть **один** путь: `...\PosCounter.Net\SpecGrid\SpecGridService.cs`

**Авто-ремонт (если файл склеен дважды внутри одного .cs):**

```powershell
cd "...\PosCounter.Net"
powershell -ExecutionPolicy Bypass -File .\build\repair-duplicate-specgrid.ps1
powershell -ExecutionPolicy Bypass -File .\build\verify-no-duplicate-sources.ps1
```

В файле — комментарий `POSC-SINGLE-FILE-SVC` и **~2206 строк**.

Если в `TableGrid.cs` тоже есть `class SpecGridService` — удалите вставленный блок в конце `TableGrid.cs` (оставьте только `TableGridBuilder`).

**Чистая перекопировка** — как в разделе PaletteHost выше.

Подробнее: `docs\BUILD.md` в полном репозитории.

## Диагностика (если спецификация не работает)

1. Уберите `(load "pos_counter_2016_2026")` из `acad.lsp`.
2. **NETLOAD** → **ЗАПУСТИТЬ** → **Выбрать спецификацию**.
3. Скопируйте из CMD все строки **`[POSC]`** и **`[POSC-DIAG]`** и пришлите разработчику.

Подробная расшифровка: `docs\INSTRUCTION_ENGINEER.md` §8.1.

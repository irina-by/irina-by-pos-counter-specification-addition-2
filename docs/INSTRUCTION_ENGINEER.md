# POS COUNTER — инструкция для инженера

Простая памятка: что нажимать в палитре и что происходит на чертеже.

**Версия палитры:** POS COUNTER v4.2.0  

**Какая DLL:**


| AutoCAD     | Файл для NETLOAD              |
| ----------- | ----------------------------- |
| 2016 – 2024 | `dll 2016\PosCounter.Net.dll` (+ `System.ValueTuple.dll` в той же папке) |
| 2025 – 2026 | `dll 2026\PosCounter.Net.dll` |


**Важно для AutoCAD 2016 (в т.ч. SP1, R20.1, .NET 4.5.2):** в папке загрузки должны лежать **два** файла — `PosCounter.Net.dll` и `System.ValueTuple.dll`. Скопируйте оба из `bin\x64\Release\net452\` после сборки в Visual Studio.

**Не используйте** старый LISP `pos_counter_2016_2026.lsp` и команду **LOAD** — только **NETLOAD** .NET DLL.

**Уберите из `acad.lsp`** строку `(load "pos_counter_2016_2026")` — иначе при каждом запуске AutoCAD: `сбой при выполнении LOAD`. На NETLOAD это не влияет, но мешает в CMD.

---

## 1. Один раз в начале смены

1. Откройте нужный чертёж.
2. В командной строке AutoCAD введите только: `**NETLOAD`**
3. Укажите файл: `**PosCounter.Net.dll**` из `dll 2016\` (AC 2016–2024) или `dll 2026\` (AC 2025+).
4. После загрузки в CMD появится строка `[POSC] PosCounter.Net … (net452) build=ГГГГ-ММ-ДД ЧЧ:ММ загружен` или `(net8.0-windows)` — **палитра откроется сама**. Дополнительно ничего вводить **не нужно**.

**Как убедиться, что загружена свежая DLL (важно для AC 2016):**

1. Соберите в Visual Studio: **Release | x64 | net452** (см. `PosCounter.Net\build\СБОРКА_VS_AC2016.md`).
2. Скопируйте оба файла из `bin\x64\Release\net452\` в `dll 2016\`.
3. Закройте AutoCAD или сделайте **NETLOAD** заново из `dll 2016\`.
4. В CMD при NETLOAD смотрите **`build=…`** — дата/время должны совпадать с только что собранной DLL.
5. В `[POSC-DIAG]` после спецификации для DBText в шапке: **`AlignmentPoint`** / **`AlignmentPoint+ExtentsTop`**, не `ExtentsCenter`.
6. Если `build=` старый — вы загрузили не ту DLL; повторите копирование и NETLOAD.

Если палитру закрыли — в командной строке можно снова набрать `**POSC`** (это единственная команда программы). Обычно достаточно снова нажать **NETLOAD** в новой сессии AutoCAD.

**В командной строке программу вручную больше не вызывают** — только кнопки в палитре.

---

## 2. Две задачи программы


| Задача              | Откуда данные                                       | Что меняется на чертеже                |
| ------------------- | --------------------------------------------------- | -------------------------------------- |
| **Подсчёт выносок** | Тексты и блоки на плане (выноски с номером позиции) | **Ничего** — только таблица в палитре  |
| **Спецификация**    | Таблица в чертеже (LINE + текст или AutoCAD Table) | Только колонка **«Кол.»** (количество) |


Связь между ними — **номер марки** (1, 2, 52…): в палитре после подсчёта есть марка и количество; после «Выбрать спецификацию» подставляется **наименование** из таблицы и в **«Кол.»** на чертеже пишется количество из палитры.

---

## 3. Кнопки вверху палитры

### ЗАПУСТИТЬ

**Что делает:** считает позиции на чертеже и заполняет таблицу в палитре (Марка, Количество, Слой).

**Как пользоваться:**


| Ситуация                                      | Действие                                                                         |
| --------------------------------------------- | -------------------------------------------------------------------------------- |
| Нужно посчитать **только выделенные** выноски | Выделите объекты на чертеже → нажмите **ЗАПУСТИТЬ**                              |
| Нужно посчитать **всё**                       | Включите галочку **«Все объекты в модели»** → **ЗАПУСТИТЬ** (выделение не нужно) |
| Ничего не выделено, галочка **выключена**     | Программа попросит выделить объекты или включить галочку                         |


**После нажатия:** внизу палитры статус вроде «Подсчёт…» / «[OK]»; в таблице появятся строки с марками и количеством.

#### Работа на листе (layout)

1. Откройте нужный **лист** (Лист4, Лист5…), не вкладку «Модель».
2. **Дважды щёлкните** внутри видового экрана — внизу должно подсветиться **MODEL**.
3. **Рамкой** выделите план или таблицу → **ЗАПУСТИТЬ**.
4. Альтернатива без рамки: галочка **«Все объекты в модели»** — подсчёт всего, что видно в активном видовом экране.

В CMD после ЗАПУСТИТЬ: `[POSC-DIAG] pick ctab=Лист… cvport=… count=…` — если `count=0`, выделение не попало в программу (проверьте шаг 2).

#### Чертежи СПДС (nanoCAD СПДС, proxy)

При открытии DWG может появиться предупреждение **SPDSExtApp** — модуль СПДС не установлен. Программа читает **графику** proxy через `Explode`, если:

1. **PROXYGRAPHICS = 1** (Настройки → Открытие и сохранение → отображать прокси-графику).
2. В CMD после ЗАПУСТИТЬ / спецификации: `[POSC-DIAG] proxy picked=… exploded=ok lines=… texts=…`.

Если `explode-fail` — включите PROXYGRAPHICS или установите **СПДС Extension for AutoCAD**. Статус палитры подскажет то же.

### Выбрать спецификацию

**Что делает:**

1. Просит **рамкой** выделить объекты **таблицы спецификации** (линии + текст; или только объект **AutoCAD Table** без линий).
2. Можно выбрать **несколько таблиц подряд** (первая, вторая…).
3. Чтобы **закончить** выбор — нажмите **Enter без выделения** (пустая рамка).
4. Программа распознаёт сетку, подставляет **наименования** в палитру и записывает **количество** в колонку **«Кол.»** на чертеже.

**Важно — порядок выделения таблиц:**

1. **Таблица 1** — лист **с шапкой** (подписи **Поз.**, **Наименование**, **Кол.** в рамке). Программа сохраняет эталон столбцов (`[SCHEMA] locked Mark=0 Name=2 Qty=3`).
2. **Таблицы 2, 3…** — листы **«Продолжение»** (шапка в рамке **не обязательна**). Рамка на **весь лист** (~200+ объектов, не фрагмент ~26). Столбцы с первой таблицы (`[SCHEMA] inherited` или `aligned`). При малой рамке: **«Рамка продолжения слишком мала»**.
3. Если первая рамка — сразу «Продолжение» без шапки, а дальше ещё таблицы — в CMD: **«Шапка не найдена в первой таблице»**.

**Ещё:**

- Сначала обычно нужен **ЗАПУСТИТЬ**, чтобы в палитре были **количества** по маркам.
- Рамка на **всю таблицу** (линии + тексты + **все** марки). Таблица 1 — ориентир ~400–550 объектов; **продолжение** — ~200–250 объектов (не фрагмент ~60 объектов).
- В CMD: `[SCHEMA] locked` / `inherited` / `aligned`, «KeyToRowMark…», «WriteQty итог».
- Если наследование не удалось: `[SCHEMA] inherit-fail reason=…` и при малой рамке — **«Рамка продолжения слишком мала»**; затем может сработать `inherit-fail → fallback infer-data`.
- Если для марки нет qty в палитре — **«Количество не найдено в палитре для марок: …»**.

**Примечания инженера в ячейке «Кол.»** программа **не удаляет**; рядом появляется или обновляется **цифра количества** в стиле основного текста таблицы.

---

### Сброс

Очищает результаты подсчёта и спецификации в палитре **без NETLOAD**. После сброса снова **ЗАПУСТИТЬ** → **Выбрать спецификацию**.

---

### Галочка «Все объекты в модели»

Влияет только на **ЗАПУСТИТЬ** (подсчёт выносок), не на спецификацию.

---

## 4. Фильтры (вторая строка палитры)

Четыре фильтра: **Марка**, **Наименование**, **Количество**, **Слой**.

- Сужают **видимые строки** в таблице; подсчёт заново **не делают**.
- Фильтр **Слой** (и остальные фильтры палитры) сужают **видимые строки**; количества для записи в **«Кол.»** на чертеже берутся из **тех же видимых** строк (как в таблице палитры).

---

## 5. Таблица в палитре


| Колонка          | Смысл                                          |
| ---------------- | ---------------------------------------------- |
| Галочка          | Для экспорта и «Показать на чертеже»           |
| **Марка**        | Номер позиции с выноски                        |
| **Наименование** | Из спецификации; можно **править вручную** в ячейке (сохраняется при повторном «Выбрать спецификацию» для этой строки) |
| **Количество**   | Сумма по выноскам                              |
| **Слой**         | Слой объекта на чертеже                        |


**Копирование:**

- **Ctrl+C** — скопировать видимые (или выделенные) строки в Excel (4 колонки).
- **Ctrl+Shift+C** — только наименования.

---

## 6. Кнопки внизу палитры


| Кнопка                  | Действие                                                                          |
| ----------------------- | --------------------------------------------------------------------------------- |
| **Показать на чертеже** | Подсвечивает на плане объекты строк с галочкой и/или выделенных в таблице (синим) |
| **Экспорт всех**        | Excel/CSV: Марка, **Наименование**, Количество, Слой (включая ручные правки)   |
| **Экспорт выделенных**  | Только строки с галочкой                                                          |
| **Экспорт видимых**     | Только то, что видно после фильтров                                               |


Строка **статуса** внизу — подсказки: готово, ошибка, сколько записано «Кол.» и т.д.

---

## 7. Рекомендуемый порядок работы

```text
NETLOAD  →  палитра открылась
    ↓
ЗАПУСТИТЬ  (выноски → марки и количества в таблице)
    ↓
(по желанию) фильтр «Слой»
    ↓
Выбрать спецификацию  (рамки таблиц → имена в палитре, цифры в «Кол.» на чертеже)
    ↓
(по желанию) экспорт или «Показать на чертеже»
```

---

## 8. Чего программа не делает

- Не меняет на чертеже марки и наименования в спецификации — только **«Кол.»**.
- Не считает **мультивыноски MLEADER** как отдельный тип (нужен обычный текст/атрибут блока).
- Не требует ввода служебных команд — всё через палитру после **NETLOAD**.

---

## 8.1. Диагностика AC 2016 (`[POSC-DIAG]`)

Если на **AutoCAD 2016** спецификация не заполняется (нет «Кол.» / наименований), снимите лог из командной строки:

1. Уберите из `acad.lsp` строку `(load "pos_counter_2016_2026")` — иначе в начале сессии будет ошибка LOAD.
2. **NETLOAD** → `dll 2016\PosCounter.Net.dll` (рядом `System.ValueTuple.dll`).
3. **ЗАПУСТИТЬ** → **Выбрать спецификацию** (тот же чертёж).
4. Скопируйте **весь** текст CMD, где есть **`[POSC]`** и **`[POSC-DIAG]`**.

Что означают ключевые строки `[POSC-DIAG]`:

| Строка | Смысл |
|--------|--------|
| `pass1: ColMark=… ColQty=…` | Что программа нашла в шапке на первом проходе |
| `fallback «столбцы по данным»` | Шапка не распознана — столбцы угаданы по цифрам в таблице |
| `inference ColQty scores` | Почему не найден столбец «Кол.» по тексту шапки |
| `ColQty=N источник=numeric` | Столбец «Кол.» угадан по числам в таблице (типично AC 2016) |
| `ColQty=N источник=simple01` | Найден по строкам 0–1 таблицы (как в старой версии) |
| `ColQty=N источник=allTexts` | Найден текст «Кол.» вне ячеек шапки |
| `ColQty=N источник=dbTextBand` | Шапка TEXT/MText в полосе GridYs (типично AC 2016) |
| `DBText в полосе шапки (GridYs)` | Какие подписи нашли в зоне шапки по сетке |
| `ColQty fallback: simple01=… numeric: col4 …` | Почему не нашли «Кол.» (cell vs text по столбцам) |
| `build=ГГГГ-ММ-ДД` в NETLOAD | Метка свежей сборки DLL |
| `вне сетки: N/M` | Тексты не попали в ячейки (частая причина на AC 2016) |
| `unassigned→name col=2 row=…` | MText вне сетки привязан к столбцу «Наименование» по DataX |
| `Количество не найдено в палитре для марок: …` | Марка **в таблице**, но **нет qty после ЗАПУСТИТЬ** — не ошибка записи; сделайте подсчёт |
| `[NAME] skip=designation/section/not-acceptable` | Почему строка не попала в наименование (по бюджету проблемных ключей) |
| `имена: … пустых=…` | Сколько наименований удалось прочитать |
| `WriteQty пропущен: ColQty=-1` | «Кол.» не пишется — столбец не найден; проверьте `build=` и пересборку |
| `WriteQty key=1 colQty=3 qty=1 Y=… (палитра)` | Запись qty из палитры в col3, верхняя суб-строка марки |
| `WriteQty итог: записано=N` | Успех: N ячеек «Кол.» заполнено на чертеже (N>0) |
| `ИТОГ: … источник=layout-evidence layout=4→3` | ColQty исправлен по evidence (масса/числа), не жёсткая схема 0/2/3 |
| `ИТОГ: … ВНИМАНИЕ: ColQty не найден` | Старая DLL или таблица без схемы — пересоберите и NETLOAD заново |
| `имя key=4 col2-lines=…` | Диагностика имён ГТ-20 (без bleed из col1) |
| `[NAME] nameLeadCap=… supplementCyrillic=…` | Граница многострочного имени; дочитана русская строка |
| `[SCHEMA] locked` после pass1=-1 | Шапка найдена на pass2 (схема 0/2/3) |
| `DLL net452 v… build=…` | Первая строка при «Выбрать спецификацию» — какая DLL в памяти |
| `[COLQTY] TryPrefer … standard=True` | Схема 0/1/2/3 — должен следовать `layout fix →3` |
| `[COLQTY] layout fix 4→3 reason=mass-column` | ColQty переназначен с доказательством (не mandatory 0/2/3) |
| `[HEADER] col2 «…» scores qty=0` | Текст шапки по столбцам; score=0 — данные вместо «Кол.» |
| `[NAME] key=1 cellOnly=true` | Имя только из CellText col2 (без bleed AllTexts) |
| `[WRITEQTY] key=1 update DBText targetY=… entY=…` | Куда пишется qty; entY близко к targetY — обновлена верхняя суб-строка |
| `[WRITEQTY] key=1 skip-far-entY … create DBText` | Старая цифра на второй строке пропущена — создан новый текст в col3 |
| `[NAME] key=52 empty …` | Пустое имя в конце таблицы — причина в cellOnly/rowEndEx |
| `[SCHEMA] locked scope=0 Mark=0 Name=2 Qty=3` | Эталон столбцов с первой таблицы (шапка найдена) |
| `[SCHEMA] inherited scope=1 …` | Продолжение: столбцы эталона, те же индексы |
| `[SCHEMA] aligned scope=1 … dx=…` | Продолжение: столбцы выровнены по X-полосам (таблица смещена на чертеже) |
| `[SCHEMA] inherit-fail reason=X-mismatch` | Сетка продолжения не совпала с эталоном — проверьте полную рамку |
| `[SCHEMA] inherit-fail → fallback infer-data` | Запасной путь: столбцы по данным (как раньше) |
| `Рамка продолжения слишком мала (N объектов)` | Выделена часть таблицы — рамка на весь лист «Продолжение» |
| `(столбцы от эталона)` в «Распознана шапка» | Таблица 2+ без поиска шапки |
| «Шапка не найдена в первой таблице» | Первая рамка без Поз./Наимен./Кол. — выберите другой лист |
| `Распознана шапка … «— (продолжение)»` | Таблица без шапки в рамке — нормально для листов «Продолжение» |
| Нет `resolved numeric col=0` до inference | Ранний numeric в col0 (марки) больше не должен появляться |

**Копирование:** Yandex `PosCounter.Net` → OneDrive → VS Release x64 net452 → NETLOAD из `bin\x64\Release\net452\`.

Файлов на диск программа **не создаёт** — только CMD.

## 8.2. Чеклист KV после NETLOAD (AC 2016)

После **Выбрать спецификацию** проверьте в CMD:

1. `DLL net452 … build=…` — свежая сборка.
2. `Палитра qty: ключей=N` — был **ЗАПУСТИТЬ**.
3. Для каждой таблицы — строка **`[KV-SUMMARY]`** (всегда, вне лимита бюджета).
4. **Шапка:** `headerPath=gridTokens` (не `topBand fallback` на малых таблицах).
5. **Марки:** `KeyToRowMark count` > 0; при «Поз.N» — `[MARK] parse … prefix=`.
6. **Имена:** `names` близко к `keys`; при `empty>0` — `[NAME] empty` и `skip=…`.
7. **Вне сетки:** `outside` минимально; есть `unassigned→name` / `[GEO]` при сдвинутых MText.
8. **Qty:** `WriteQty итог записано>0`; missing — только отсутствие в палитре.
9. Скопируйте **весь** блок `[POSC-DIAG]` + `[KV-SUMMARY]` для анализа.

Категории: `[HEADER-SCAN]`, `[MARK]`, `[GEO]`, `[NAME]`, `[COLQTY]`, `[WRITEQTY]`, `[SCHEMA]`, `[KV-SUMMARY]`.

## 8.3. Палитра vs выбранные таблицы

Если после **ЗАПУСТИТЬ** в палитре N марок, а после **Выбрать спецификацию** имён меньше (M < N):

1. В CMD: `[POSC] Палитра: ключей=N, имён из выбранных таблиц=M`.
2. Строка `Без имени (нет на выделенных листах): …` — марки, которые есть в палитре qty, но **не на выделенных** листах.
3. **Решение:** выделите **все** листы/таблицы спецификации на чертеже (или только те, где нужны имена).
4. В статусе палитры: `имён=M из N ключей`.

Дополнительно для имён и qty:

| Паттерн CMD | Значение |
|-------------|----------|
| `[NAME] reason=merged-block` | cellOnly отключён — многострочный merged-блок |
| `[NAME] reason=missing-cyrillic` | добавлен AllTexts pass для RU+EN |
| `[WRITEQTY] from=colName-rowTop` | стиль qty как у «Наименование» на rowTop |
| `grep SpecGrid` без `key ==` | нет хардкода марок |

## 9. Если что-то пошло не так


| Проблема                       | Что проверить                                                     |
| ------------------------------ | ----------------------------------------------------------------- |
| Палитра не открылась           | Снова **NETLOAD** или команда **POSC**                            |
| Пустая таблица после ЗАПУСТИТЬ | Выделение / галочка «Все объекты»                                 |
| «Кол.» не заполнилось          | Был ли **ЗАПУСТИТЬ**; видна ли марка в палитре после фильтра слоя |
| Сообщение «Заполните колонку Марка» | В таблице в **«Поз.»** должны быть чистые цифры 1, 2, 3… |
| Сообщение «Распознана шапка» | В CMD видно, какие столбцы программа нашла (Марка/Наименование/Кол.). Если «Кол.» не найдена — поправьте заголовок и/или строку шапки |
| «Сетка: линии на разных слоях» | Горизонтали и вертикали на разных слоях — программа дополняет сетку; лучше свести линии на один слой |
| Цифра не в «Кол.» / огромная   | Обновите DLL; проверьте рамку выбора и заполнение «Поз.»        |
| Нет наименования (марка 1)     | В CMD: `RowDataStart`, `KeyToRowMark: 1→row1`; цифра «1» в «Поз.» с первой строки данных |
| Нет наименования               | Слой строки отфильтрован; таблица без шапки в рамке — **Сброс**, **ЗАПУСТИТЬ**, рамка с «Поз.»/«Наименование»; или введите имя вручную в палитре |
| Распознавание «то да, то нет»  | **Сброс** → NETLOAD → повтор; в CMD смотрите `gridLayer`, `(столбцы по данным)` |
| AC 2016: шапка не найдена      | **Сброс** → **ЗАПУСТИТЬ** → рамка **с шапкой**; в `dll 2016\` оба DLL; в CMD — `(net452)` после NETLOAD |
| AC 2016: «Кол.» пусто после этапа 2 | Пересоберите `build-ac2016.cmd`; в CMD: `ColQty=3 источник=layout`, `WriteQty итог: записано>0` |
| Сборка: `System.ValueTuple 4.5.0 не найден` | Перекопируйте `PosCounter.Net` с Yandex (папка `lib\netstandard1.0\`); VS **Очистить** → **Собрать** |
| ГТ-20: qty 6 вместо 1 | Старая DLL или `ColQty=-1`; нужны `layout=→3` и `WriteQty key=1 qty=1` |
| Сборка на ПК инженера | Скопировать папку из Yandex в OneDrive → VS Release x64 net452 → `bin\x64\Release\net452\PosCounter.Net.dll` → NETLOAD |
| AC 2016: нужна диагностика     | После **Выбрать спецификацию** скопируйте из CMD все строки **`[POSC-DIAG]`** и **`[POSC]`** и пришлите разработчику |
| Ошибка LOAD `pos_counter_…`    | Уберите `(load …)` из `acad.lsp` / `acaddoc.lsp`; только NETLOAD |
| Старая версия                  | Закройте AutoCAD, замените DLL в `dll 2016` или `dll 2026`, снова **NETLOAD**    |

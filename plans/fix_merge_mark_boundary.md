## Проблема (до фикса)

На больших двуязычных таблицах имя **следующей** марки попадало в **предыдущую** (bleed):

| Марка | Симптом в палитре |
|-------|-------------------|
| 5 | в конце текст марки 6 («фланцевый ∅65») |
| 6, 7, 10, 11 | аналогично — текст следующей марки |

Шапка и `WriteQty` работали; bleed — только в **наименованиях**.

## Корневая причина

В `ResolveNameForKey` для merged-блоков:

```csharp
// УБРАНО (2026-06-17):
rowEndExclusive = Math.Max(rowEndExclusive, Math.Min(nextMarkRow + 1, gridRows));
```

Граница шла до **цифры** следующей марки, захватывая RU/EN строки **над** цифрой.

## Этап 2 — реализация

| Метод | Назначение |
|-------|------------|
| `ResolveNextMarkBoundaryExclusive` | `min(nextKeyTop, nextMarkRow)` когда оба > `rowTop` |
| `FinalizeMarkBlockEndExclusive` | тот же cap для `KeyToMarkBlockEnd` |
| `ResolveNameForKey` | `rowEndExclusive = min(markBlockEnd, boundary)` |
| `ReportMergeBoundaryBleedWarnings` | `[POSC] ВНИМАНИЕ …` только если bleed остался |

Подробности: `docs/DEVELOPER.md` §11.

## Проверка в AutoCAD (инженер)

**Сборку делает инженер вручную** (`build-ac2016.cmd` или VS) → NETLOAD.

1. ЗАПУСТИТЬ → Выбрать спецификацию на **028_МВ_AR-rev2**
2. Палитра: марка **5** — только кран ∅15 угловой, **без** фланцевого ∅65
3. Марки **6, 7, 10, 11** — без текста следующей марки
4. CMD: **нет** `[POSC] ВНИМАНИЕ Табл.… марка …`
5. Малая таблица (2–4 строки в марке) — без регрессии

**Отдельно:** список «Количество не найдено в палитре для марок: …» — нет qty после ЗАПУСТИТЬ; не связано с bleed.
